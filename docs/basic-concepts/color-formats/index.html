Color Formats

The possible colors of an application has an impact on many parts of the application. From the visual appearance to memory consumption and overall performance. This section will detail the color format choices available in TouchGFX and highlight pros and cons and where they apply.

#Color depth

Every pixel has a color. In TouchGFX such a color is represented by a number. Color depth describes the number of possible colors. The color depth is measured in bits per pixel (bpp).

  • A color depth of 1 bpp implies that only two colors can be represented. Zero and one. Typically black and white.
  • A color depth of 2 bpp implies that four different colors can be represented. One through three.
  • ...
  • A color depth of 24 bpp implies that (2^24) 16,777,216 different colors can be represented.
