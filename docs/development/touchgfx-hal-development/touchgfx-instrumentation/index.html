### Measuring Performance

The TouchGFX Engine is instrumented with specific calls to an interface, GPIO, at important stages in the rendering lifecycle which allows users to measure the performance of an application. The enum GPIO_ID reflects the types of signals being toggled. Only the signals RENDER_TIME and FRAME_RATE are used internally by the TouchGFX Engine using the functions, set, clear and toggle.

class GPIO
{
public:


    typedef enum
    {
        VSYNC_FREQ,  /// Pin is toggled at each VSYNC
        RENDER_TIME, /// Pin is high when frame rendering begins, low when finished
        FRAME_RATE,  /// Pin is toggled when the frame buffers are swapped.
        MCU_ACTIVE   /// Pin is high when the MCU is doing work (i.e. not in idle task).
    } GPIO_ID;


    static void init();
    static void set(GPIO_ID id);
    static void clear(GPIO_ID id);
    static void toggle(GPIO_ID id);
    static bool get(GPIO_ID id);
};
