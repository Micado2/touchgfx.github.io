Rendering

#TouchGFX main loop

In pseudo code the main loop inside TouchGFX looks like

while(true) {
    wait();
    update();
    render();
}

The code is much more involved in the real implementation, but the pseudo code above helps in understanding the main parts of the engine.

#Wait

TouchGFX will wait until it is required to draw a frame.

The waiting is over when TouchGFX is informed from the outside world that it should proceed. Typically this information comes in the form of a

  • synchronization interrupt from MCU or display
  • hardware timer interrupt
