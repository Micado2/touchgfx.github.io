Display

This chapter focuses on the considerations that should be included when selecting the right display for your embedded GUI product.

#Display Interface Overview

The display is connected to the MCU via different types of interfaces. The interface varies on different parameters, and below section is addressing some graphics related ones like, number of pins needed, max bandwidth supporting different resolutions.

#Brightness & Backlight

Often measured in Candelas/M2. Backlights are the most power consuming part of the display. In sunlight one would need around 600 cd/M2. Often higher brightness gives higher heat minimizing the live time of the LEDs.

#Viewing Position and Color Inversion

When embedding a display into a product, it is important to anticipate and know which viewing positions the user can have. In some displays from certain viewing positions, a color inversion can happen. So installing the display in the right position, allowing the user to operate and experience the GUI while seeing the right colors designed by the graphics designer can be tricky.

The color inversion can happen on TN panels. Adding a SWV film can help increasing viewing angles.

Viewing position

#Display Lifetime

The lifetime is defined as the time until the display reached half brightness at 25 degrees. So if your product needs to last for 10 years, running a high amount of hours pr. day, then this parameter could be relevant.

#Pixel density

Pixel density is defining how many pixels is shown pr. Inch or square inch. Choosing the right pixel density can depend on the expectations from the end user, environment, design needs etc. Putting this into perspective a HUAWEI P30 running a 6.1” 23401080 has a pixel density pr. square inch of 178,500, and a commonly used 5” TFT display running 800480 has 34.816 Pixels pr. Square inch.

#Dithering

In the above picture, Floyd-steinbreg dithering is applied. Meaning a dithering algorithm is added to create the illusion of a higher pixel depth (24 bpp), being able to show colors not originally represented in the 24 bit per pixel image (16,777,216 colors).

#Dynamic color range

Is in this case the ratio between two contrast colors, like black and white. In the above example, the eye of the parrot there contains different levels of white and black. The image on the left has PPI2 at 54,000 vs 17,763 on the right, this means that the display on the left has more pixels to show all the colors represented, creating a smoother transition between different colors.

#Anti-aliasing

When resolutions are too low, a staircase effect can appear. Using anti-aliasing in the application can smooth out these staircase edges in an image. When looking close at the display, the staircase effect slightly appears on the right image, as the pixel density does not allow the display to represent a high enough color range enabling high enough anti-aliasing.
