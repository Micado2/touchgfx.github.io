virtual

Blits a 2D source-array to the framebuffer perfoming alpha-blending (and tranparency keying) as specified Performs a software blend if HAL does not support BLIT_COPY_WITH_ALPHA and alpha != 255.

Parameters
sourceDataThe source-array pointer (points to the beginning of the data). The sourceData must be stored as 16-bits RGB565 values.
sourceThe location and dimension of the source.
blitRectA rectangle describing what region is to be drawn.
alphaThe alpha value to use for blending (255 = solid, no blending)
hasTransparentPixelsIf true, this data copy contains transparent pixels and require hardware support for that to be enabled.
